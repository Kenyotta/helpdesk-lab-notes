# Legal Name Change / Active Directory Account Update Guide

## Overview

This guide covers requests where a user has undergone a legal name change (e.g., marriage) and requires updates to their Active Directory account, display name, and email address.

---

## Common Scenarios

* Employee recently married and changed last name
* HR submits official name change request
* User requests updated email address
* Request to maintain old email for continuity

---

## Required Verification

Before making any changes:

* Confirm HR approval is documented
* Verify user identity through internal verification process
* Confirm requested changes match HR records

---

## Account Elements Affected

### 1. Display Name

* Update to reflect legal name change
* Used in email display and internal directories

### 2. Primary Email Address

* Updated to match new legal name (if required)
* Becomes main email identity

### 3. Email Alias (Important)

* Keep previous email address as alias
* Ensures no missed emails from old contacts

### 4. Username

* Typically NOT changed
* Prevents login issues and broken group memberships

---

## Troubleshooting / Execution Steps

### Step 1: Locate User Account

* Search Active Directory using current username or old email

---

### Step 2: Verify Authorization

* Confirm HR approval before proceeding

---

### Step 3: Update Display Name

* Change to new legal name format

---

### Step 4: Update Email Address

* Set new primary email address if required

---

### Step 5: Preserve Legacy Email

* Add previous email as alias
* Confirm mail routing remains active

---

### Step 6: Validate Changes

* Ensure account still logs in correctly
* Confirm email routing functions for both addresses

---

## Common Issues

### Issue: User cannot log in after update

* Likely caused by username change (avoid changing username)

### Issue: Emails not received from old address

* Alias not properly configured

### Issue: HR mismatch

* Changes made without HR approval

---

## When NOT to Escalate

* Standard name change request with HR approval
* No login or system errors reported
* Only display/email update required

---

## When to Escalate

* HR approval missing or unclear
* Login issues after update
* Email routing failures across systems
* Multiple system identity mismatches

---

## Key Takeaway

Legal name changes are **identity management tasks**, not technical troubleshooting issues. The goal is to ensure consistency across systems while maintaining uninterrupted communication through aliases.
